# Homework: OOP — Product & Category

## 📌 Описание проекта
Учебный проект, реализующий базовую объектно-ориентированную модель для интернет-магазина. В проекте созданы два класса: `Product` (товар) и `Category` (категория товаров).

## 🚀 Функционал
- **Класс `Product`** — хранит информацию о товаре: название, описание, цену и количество на складе.
- **Класс `Category`** — хранит информацию о категории: название, описание и список товаров. Автоматически подсчитывает общее количество категорий и товаров во всех созданных объектах через атрибуты класса `category_count` и `product_count`.
- **Тестирование** — полное покрытие функционального кода тестами с использованием `pytest`.
- **Отчёт о покрытии** — генерируется с помощью `pytest-cov` в текстовом и HTML-форматах.

- pytest — тестирование
- pytest-cov — измерение покрытия кода тестами

# Inventory Management System

Система управления инвентарём магазина с поддержкой абстрактных классов и миксинов.

## Функциональность

### Архитектура классов
- **BaseProduct** — абстрактный базовый класс (ABC) с абстрактными методами `__str__`, `__add__`, `__repr__`
- **InfoMixin** — класс-миксин, печатающий информацию о создании объекта через `__repr__`
- **Product** — базовый класс продукта, наследует `InfoMixin` и `BaseProduct`

### Классы-наследники
- **Smartphone** — наследник Product с атрибутами: производительность, модель, память, цвет
- **LawnGrass** — наследник Product с атрибутами: страна, срок прорастания, цвет

### Магические методы
- `__str__` — строковое представление продуктов и категорий
- `__repr__` — официальное представление для отладки
- `__add__` — сложение продуктов (только одинаковых классов)
- `__iter__` / `__next__` — итерация по товарам категории

### Ограничения
- Нельзя создать экземпляр BaseProduct (абстрактный класс)
- Сложение товаров разных классов вызывает `TypeError`
- Добавление в категорию только объектов Product или его наследников

- Отчёт о покрытии: `test_results/`
